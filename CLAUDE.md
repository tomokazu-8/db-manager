# DBトリッジ マネージャー — プロジェクト引継ぎ

## 用語定義

- **トリッジ（Tridge）**: 見積アプリに差し込むExcelファイル。"cartridge"から命名。
  - 旧称「DBカセット」→ 現在は「トリッジ」に統一
  - アプリ内UIの表記もすべて「トリッジ」に変更予定

## このアプリの目的

八友電工の見積アプリ（estimate-app）で使用する「トリッジ」（Excelファイル）を
作成・編集・管理するための独立したWebアプリ。

## 親プロジェクトとの関係

```
db-manager（このアプリ）       estimate-app（見積プラットフォーム）
  トリッジを作成・編集  →  Excelエクスポート  →  見積アプリでトリッジ装着
```

- estimate-app のリポジトリ: `c:\Users\pal19\Projects\estimate-app`
- GitHub Pages: https://tomokazu-8.github.io/estimate-app/

## トリッジ運用の背景

estimate-appは「見積プラットフォーム」として汎用化する方針。
- 電気設備・空調設備・給排水・建築など、業種に応じてトリッジを差し替えて使う
- 工種カテゴリ・自動計算ルール・材料マスタがすべてトリッジで定義される
- 内蔵DBは廃止済み（data/material_db.json, data/bukariki_db.json は空の[]）

## トリッジのExcel構成（設計済み・Phase 1で実装予定）

```
トリッジ（Excel）
├── 資材マスタ     ← 現在実装済み
├── 工種マスタ     ← Phase 1で追加（工種カテゴリを定義）
├── 設定マスタ     ← Phase 1で追加（アプリ動作パラメーター）
└── 労務単価マスタ  ← 将来実装予定
```

### 資材マスタ（現在実装済み）
列順：品目名称 / 規格名称 / 単位 / 基準単価 / 原価単価 / 原価率 / 歩掛1 / 中分類名 / カテゴリ

### 工種マスタ（Phase 1で追加予定）
| 列名 | 内容 | 例 |
|-----|------|-----|
| 工種ID | 英数字のID（estimate-app内部で使用） | `trunk` |
| 工種名 | 見積書に表示される名称 | `幹線・分電盤工事` |
| 略称 | タブ等の短縮表示 | `幹線・分電盤` |
| 割合モード | 1=金額を割合で自動計算、0=通常 | `0` |
| 雑材料率% | 材料費に対する雑材料の割合 | `5` |
| 順序 | 表示順（数値） | `1` |

### 設定マスタ（Phase 1で追加予定）
| パラメーター | 値 | 説明 |
|------------|-----|------|
| 銅建値補正 | 有効 | 有効/無効（電気・空調トリッジのみ有効） |
| 銅建値基準（円/kg） | 1000 | トリッジ作成時点の基準値 |
| 銅連動率 | 0.50 | 材料価格に占める銅コスト比率 |
| 労務売単価（円/人工） | 33000 | |
| 労務原価単価（円/人工） | 12000 | |

## 現在の実装状態

### 完成済みファイル
- `index.html`    — 画面レイアウト（サイドバー + メインテーブル + モーダル）
- `css/style.css` — デザイン（estimate-appと同じデザイントークン使用）
- `js/data.js`    — 定数・norm()・localStorage操作・ユーティリティ
- `js/app.js`     — 全UIロジック（レンダリング・イベント・Excel/CSV入出力）
- `AI変換指示書.md` — 任意形式データをCSVに変換するAI向け指示書

### 現在の機能
1. **新規トリッジ作成** — 名称・メモを入力して空のトリッジを作成
2. **Excel / CSVインポート** — .xlsx / .xls / .csv を取り込んで新規トリッジとして登録
3. **Excelエクスポート** — estimate-app互換の.xlsxとしてダウンロード（資材マスタのみ）
4. **インライン編集** — テーブルセルを直接クリックして編集
5. **自動カテゴリ検出** — 品名入力時にカテゴリを自動判定
6. **原価自動計算** — 売単価×原価率→原価を自動入力
7. **絞り込み** — カテゴリ・キーワードフィルタ
8. **自動保存** — 編集後500msでlocalStorageに自動保存（Ctrl+Sでも保存）
9. **トリッジ管理** — 名称変更・削除（サイドバーの✏/🗑ボタン）

### データ永続化
localStorageを使用：
- `dbm_db_list` — トリッジ一覧メタ情報の配列
- `dbm_db_data_{id}` — 各トリッジの品目データ

## Phase 1 実装タスク（次のセッションで着手）

### db-manager側（このアプリ）
1. UIのテキストを「カセット」→「トリッジ」に変更
2. 工種マスタ編集タブを追加（工種の追加・削除・順序変更・割合モード設定）
3. 設定マスタ編集パネルを追加（銅建値補正ON/OFF・各種パラメーター）
4. Excelエクスポートに工種マスタ・設定マスタシートを含める
5. localStorageのデータ構造に工種・設定を追加

### estimate-app側（別ウィンドウで作業）
1. excel-loader.jsに工種マスタ・設定マスタの読み込み処理を追加
2. data.jsのCATEGORIES固定値を削除→トリッジから動的ロード
3. 銅建値補正UIをトリッジの設定マスタで動的に表示/非表示
4. labor.jsの電気専用キーワードを汎用化（Phase 2）

## AI変換フロー

```
元データ（任意の形式）
  ↓ AI変換指示書.md を Claude に渡して変換依頼
  ↓ CSVで出力してもらう
db-manager で「Excel / CSVからトリッジ作成」で読み込み
  ↓ 内容確認・修正（インライン編集）
  ↓ Excelエクスポート（例: 新築電気設備用トリッジ.xlsx）
estimate-app でトリッジ装着
```
