# DBトリッジ マネージャー — プロジェクト引継ぎ

## 用語定義

- **トリッジ（Tridge）**: 見積アプリに差し込むExcelファイル。"cartridge"から命名。
  - 旧称「DBカセット」→ 現在は「トリッジ」に統一済み
  - アプリ内UIの表記もすべて「トリッジ」に変更済み

## このアプリの目的

八友電工の見積アプリ（estimate-app / Deck）で使用する「トリッジ」（Excelファイル）を
作成・編集・管理するための独立したWebアプリ。

## 親プロジェクトとの関係

```
db-manager（このアプリ）       estimate-app（Deck = 見積プラットフォーム）
  トリッジを作成・編集  →  Tridgeエクスポート  →  Deckでトリッジ装着
```

- estimate-app のリポジトリ: `c:\Users\pal19\Projects\estimate-app`
- GitHub Pages: https://tomokazu-8.github.io/estimate-app/

## トリッジ運用の背景

estimate-appは「見積プラットフォーム（Deck）」として汎用化する方針。
- 電気設備・空調設備・給排水・建築など、業種に応じてトリッジを差し替えて使う
- 工種カテゴリ・自動計算ルール・材料マスタがすべてトリッジで定義される
- 内蔵DBは廃止済み（data/material_db.json, data/bukariki_db.json は空の[]）

## トリッジのExcel構成（5シート構成 = Deck側コネクタ仕様に準拠）

```
トリッジ（Excel）
├── 資材マスタ       — 品目名称/規格/単位/基準単価/原価/原価率/歩掛/中分類名/カテゴリ
├── 工種マスタ       — 工種ID/工種名/略称/割合モード/雑材料率%/順序/自動計算行
├── 設定マスタ       — パラメーター名/値（銅建値補正等）
├── キーワードマスタ  — キーワード/分類/歩掛/銅連動/天井開口
└── 労務単価マスタ   — 労務区分/見積単価/原価単価
```

### 資材マスタ
列順：品目名称 / 規格名称 / 単位 / 基準単価 / 原価単価 / 原価率 / 歩掛1 / 中分類名 / カテゴリ

### 工種マスタ
| 列名 | 内容 | 例 |
|-----|------|-----|
| 工種ID | 英数字のID（estimate-app内部で使用） | `trunk` |
| 工種名 | 見積書に表示される名称 | `幹線・分電盤工事` |
| 略称 | タブ等の短縮表示 | `幹線・分電盤` |
| 割合モード | 1=金額を割合で自動計算、0=通常 | `0` |
| 雑材料率% | 材料費に対する雑材料の割合 | `5` |
| 順序 | 表示順（数値） | `1` |
| 自動計算行 | パイプ区切りの自動計算行名リスト | `雑材料消耗品\|電工労務費\|運搬費` |

### 設定マスタ（パラメーター名/値 の2列）
| パラメーター名 | 説明 |
|------------|------|
| 銅建値補正 | 有効/無効（電気・空調トリッジのみ有効） |
| 銅建値基準（円/kg） | トリッジ作成時点の基準値 |
| 銅連動率 | 材料価格に占める銅コスト比率 |
| 労務売単価（円/人工） | 労務費計算用 |
| 労務原価単価（円/人工） | 労務費計算用 |

### キーワードマスタ
| 列名 | 説明 |
|------|------|
| キーワード | 品名に含まれる文字列（norm()適用） |
| 分類 | wiring / fixture / equipment |
| 歩掛 | デフォルト歩掛係数 |
| 銅連動 | ○で銅建値補正対象（ケーブル類） |
| 天井開口 | ○で天井開口カウント対象 |

## 現在の実装状態

### 完成済みファイル
- `index.html`    — 画面レイアウト（サイドバー + タブUI + モーダル）
- `css/style.css` — デザイン（estimate-appと同じデザイントークン使用）
- `js/data.js`    — 定数・norm()・localStorage操作・ユーティリティ
- `js/app.js`     — 全UIロジック（レンダリング・イベント・Excel/CSV入出力）
- `AI変換指示書.md` — 任意形式データをCSVに変換するAI向け指示書

### 現在の機能
1. **新規トリッジ作成** — 名称・メモを入力して空のトリッジを作成
2. **Excel / CSVインポート** — .xlsx / .xls / .csv を取り込み（5シート全対応）
3. **Tridgeエクスポート** — Deck互換の5シート.xlsxとしてダウンロード
4. **タブUI** — 資材マスタ / 工種マスタ / 設定マスタ / キーワードマスタの4タブ
5. **インライン編集** — テーブルセルを直接クリックして編集
6. **自動カテゴリ検出** — 品名入力時にカテゴリを自動判定
7. **原価自動計算** — 売単価×原価率→原価を自動入力
8. **絞り込み** — カテゴリ・キーワードフィルタ（工種マスタ連動）
9. **自動保存** — 編集後500msでlocalStorageに自動保存（Ctrl+Sでも保存）
10. **トリッジ管理** — 名称変更・削除（サイドバーの✏/🗑ボタン）

### データ永続化
localStorageを使用：
- `dbm_db_list` — トリッジ一覧メタ情報の配列
- `dbm_db_data_{id}` — 各トリッジの品目データ
- `dbm_db_koshu_{id}` — 工種マスタ
- `dbm_db_settings_{id}` — 設定マスタ
- `dbm_db_keywords_{id}` — キーワードマスタ

## AI変換フロー

```
元データ（任意の形式）
  ↓ AI変換指示書.md を Claude に渡して変換依頼
  ↓ CSVで出力してもらう
db-manager で「Excel / CSVからトリッジ作成」で読み込み
  ↓ 内容確認・修正（インライン編集）
  ↓ Tridgeエクスポート（例: 新築電気設備用トリッジ.xlsx）
estimate-app（Deck）でトリッジ装着
```

## 技術スタック
